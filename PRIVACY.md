# Privacy Policy — Quick Screenshot & Annotate

**Last updated:** July 5, 2026

## Summary

Quick Screenshot & Annotate does **not** collect, transmit, or store any personal data. All processing happens entirely within your browser.

## Data Collection

**This extension collects zero data.** Specifically:

- ❌ No analytics or telemetry
- ❌ No user tracking
- ❌ No cookies
- ❌ No network requests to external servers
- ❌ No personal information collected
- ❌ No browsing history collected
- ❌ No login or account required

## Screenshots

- Screenshots are captured using Chrome's built-in `chrome.tabs.captureVisibleTab` API
- All screenshot data stays **entirely on your device**
- Screenshots are stored locally in `chrome.storage.local` (browser storage only)
- No screenshot data is ever uploaded, transmitted, or shared
- You can clear all stored screenshots at any time via the "Clear History" button in the popup

## Permissions

The extension requests the following permissions:

| Permission | Purpose |
|------------|---------|
| `activeTab` | Access the current tab to capture screenshots |
| `storage` | Save your settings (format, quality) and recent screenshot thumbnails locally |
| `scripting` | Inject the selection overlay when you choose "Capture Selected Area" |

These permissions are used **exclusively** for the extension's core functionality. No data is sent to any server.

## Third Parties

This extension does not use any third-party services, analytics, CDNs, or external resources. All code runs locally.

## Changes to This Policy

If this policy changes, the updated version will be posted here with a new date.
